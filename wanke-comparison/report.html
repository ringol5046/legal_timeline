1The three charts, rendered

The two prior charts (v1 and v2) use Mermaid's timeline block — a flat, sectioned linear list. The new chart uses D3 with swimlanes for parties and a separate fact-vs-procedure track when the case warrants it. The new chart is interactive: hover any event for source citation and jurisdictional significance.

4Structural features only the new chart encodes

  • SoL dispute as a first-class record. AVB's anchor (July 2014 license suspension) and Wanke's anchor (2013 subcontract breaches) are encoded as two competing events with date_certainty: "disputed" and linked via a Dispute record (type: meaning, resolution: adjudicated_for_wanke). The old extractor embeds this only in prose phase_summary.
  • Competing SoL clocks rendered as two tolling relationships — one with strength: "alleged" (AVB's rejected theory) and one with strength: "adjudicated" (Wanke's adopted theory). Lets the reader literally see both legal theories on one chart.
  • Lien expirations as typed tolling arrows with clock_id (levy_lien_697_710, exam_lien_708_120c) and clock_effect: "expire" — queryable, not just visual.
  • Party attribution via lanes. Each event lives on Plaintiff, Defendant, Judgment Debtors, or Court — making the who-did-what answerable at a glance. Both old charts are party-blind.
  • Per-event source citations surfaced in tooltip (Slip op. p. 7, p. 13, fn. 3 etc.). The old extractor stores grounding_refs in JSON but does not surface them in the rendered chart.
  • Auto style selection. The new chart skill picked dual-track automatically based on the 62/38 factual/procedural ratio. The old extractor renders one fixed style.

5Where the old extractor still does better

Phase grouping

  • Five named phases ("Underlying contracts", "Status changes and enforcement", "Lien expirations", etc.) provide narrative chunking ideal for executive summaries.
  • The new schema has no explicit Phase concept yet.

grounding_refs[] with verbatim quotes

  • Old JSON stores both page AND the quoted text supporting each fact.
  • New schema only stores the citation string — no quoted text. Worth adding in iteration 2.

review_flags & completeness_audit

  • Old output flags upstream extraction issues (e.g., "v1 silently merged levy with examination — de-conflated in v2") and reports completeness.
  • New skill has no equivalent metadata layer.

not_stated_notes[] & glossary_refs[]

  • Explicit field for facts the opinion is silent on; glossary terms anchored to events for layered teaching.
  • New schema covers "not stated" only implicitly via date_certainty: "unknown".

6Bottom line

Old v1 7 events

Sparse outline; useful as a one-glance executive timeline. Loses the SoL story; conflates levy with examination; misattributes amounts to procedural events.

Old v2 13 events

Comprehensive event list with proper clock-expiration tracking and corrected attributions. Still party-blind, dispute-blind, and relationship-blind — the visual format can't express them.

New 21 events

Same factual coverage as v2 plus 8 additional events (named subcontracts, collateral proceedings, AVB trial brief, Wanke's accrual theory). Critical win: the SoL dispute is visible. AVB's July 2014 license-suspension anchor and Wanke's 2013 subcontract anchor sit side by side with a red dispute bracket — the central legal question of the case is literally visible on the chart.
